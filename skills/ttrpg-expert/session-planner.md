# Session Planner: PC-Focused Prep and Character Arc Management

Structured workflows for planning sessions with dedicated
per-PC content, tracking character arcs, and balancing
spotlight time. Read when preparing a session, planning an
arc, or ensuring each PC gets meaningful content.

## PC Roster Review

Complete for every PC before planning scenes. The touchpoint
at the bottom is the critical output.

```
PC NAME: ___  PLAYER: ___

BACKSTORY HOOKS (unresolved elements from history):
CURRENT GOALS (short-term and long-term):
ACTIVE ARC STAGE (see below — current phase, next beat):
RELATIONSHIPS (key NPCs/PCs, current state):
RECENT DECISIONS (last 1-3 sessions, need consequences):
MECHANICAL HIGHLIGHTS (unused abilities waiting to shine):
LAST SPOTLIGHT SESSION: ___
SESSIONS SINCE LAST SPOTLIGHT: ___
THIS SESSION'S TOUCHPOINT (at least one moment for this PC):
```

## Five-Stage Arc Model

Character arcs follow a TV-series structure — personal
storylines weaving through the ensemble narrative.

| Stage | Duration | What Happens | GM Actions |
|-------|----------|-------------|------------|
| 1. Establishment | Sessions 1-3 | PC's beliefs, flaws, goals introduced | Record stated values. Introduce NPCs who reflect/challenge PC identity. Plant backstory seeds in world. |
| 2. Testing | Sessions 3-10+ (longest) | Situations test beliefs, strain relationships, complicate goals | Design encounters probing stated values. Moral dilemmas with no clean answer. NPCs with opposing but sympathetic views. Temptations away from stated goals. |
| 3. Crisis | 1-2 sessions (+ 2-3 escalation) | Turning point — defining choice, confront flaw, face consequences | Build pressure over 2-3 sessions. Stakes personal and irreversible. Party witnesses/participates. Do NOT predetermine outcome. |
| 4. Transformation | 1-3 sessions | Character changes from crisis — new belief, achieved hollow goal, loss, growth | Reflect change via NPC reactions and world consequences. Let player update beliefs/goals. New complications from transformation. |
| 5. New Equilibrium | 1-2 sessions | Changed state settles; becomes new Establishment for next arc | Acknowledge completed arc. Plant seeds for next arc. Use completed arc as connective tissue for other PCs. |

### Arc Tracker

```
CAMPAIGN: ___  SESSION: ___

| PC | Arc Theme | Stage | Next Beat | Target Session |
|----|-----------|-------|-----------|----------------|
```

**Arc Theme:** short phrase for core tension (e.g. "loyalty
vs. freedom", "confronting the past").
**Next Beat:** specific scene/event advancing this arc.
**Target Session:** stagger across sessions — don't advance
every arc simultaneously.

## Spotlight Planning

### A/B/C Plot Model

| Plot | Share | Description |
|------|-------|------------|
| A-Plot | 50-60% | Main campaign storyline, all PCs involved |
| B-Plot | 25-35% | Personal arc for one "featured" PC this session |
| C-Plot | 10-15% | Lighter personal touch for a second PC — brief scene, callback, arc clue |

### B-Plot Rotation

Rotate the featured PC. Track explicitly:

```
| Session | A-Plot | B-Plot (Featured PC) | C-Plot (Secondary) |
|---------|--------|----------------------|--------------------|
```

Rules:
- Every PC gets B-plot within N sessions (N = party size)
- Mid-crisis PC (Stage 3) may get consecutive B-plots
- C-plot PC in session N → B-plot candidate for N+1 or N+2

### Spotlight Tracking

```
SESSION: ___

| PC | Spotlight Share | Notes |
|----|----------------|-------|

IMBALANCE FLAG: [Yes/No]
Corrective action: ___
```

Below 15% share (in 4-player party) → corrective plan next
session. Sustained imbalance 3+ sessions is serious.

### Weaving Personal Content into Group Scenes

Avoid splitting the party for personal scenes. Instead:
- **Embed in group encounters:** group visits military
  outpost → former soldier PC's personal content emerges
- **NPCs as vectors:** backstory NPC appears in a scene
  involving the whole party
- **Multi-hook encounters:** each element appeals to a
  different PC (informant for rogue, debtor for paladin,
  rare book for wizard, hidden shrine for cleric)
- **Witness and react:** personal moments gain weight when
  other PCs are present to respond

## PC Touchpoints

Every session: at least one touchpoint per PC. Not all need
to be major — some signal the campaign remembers the character.

| Type | Impact | Description | Design Notes |
|------|--------|-------------|-------------|
| Backstory Connection | High | NPC/location/event from PC's history appears | Use sparingly: 1-2 per arc. Overuse makes world feel artificial. |
| Moral Dilemma | High | Forces choice between competing values; targets specific PC beliefs | No clean answer. If the "right" choice is obvious, it's not a dilemma. |
| Ability Showcase | Medium | Encounter designed so one PC's specific skills are key to success | Rotate across PCs. Avoid repeatedly showcasing same character. |
| Decision Callback | Medium | World reacts to a previous PC choice (2-5 sessions delayed) | Track decisions in post-session recap. Immediate feels reactive; delayed feels like world memory. |
| Arc Advancement Clue | Medium | Clue/hint advancing PC's personal arc | Ideal C-plot touchpoint. Brief, suggestive, forward-looking. |
| Character Moment | Low-Med | Quiet scene for self-expression without plot pressure (campfire, downtime) | Schedule ≥1 per session after high-tension scenes for pacing variety. |

### Touchpoint Planning Matrix

```
SESSION: ___

| PC | Touchpoint Type | Description | When in Session |
|----|----------------|-------------|-----------------|

COVERAGE CHECK:
- [ ] Every PC has at least one touchpoint
- [ ] B-plot PC has a high-impact touchpoint
- [ ] No PC without high-impact touchpoint for 3+ sessions
- [ ] Touchpoint types vary across PCs
```

## Connecting PC Arcs to Main Plot

### Web of Connections

Map during campaign planning (not session-by-session):

```
MAIN ARC: ___  ANTAGONIST: ___

| PC | Backstory Element | Connection to Main Arc |
|----|-------------------|------------------------|
```

If no natural connection: antagonist affected PC's past,
PC's organisation is a target, PC's community is threatened,
or PC has something antagonist needs/fears.

**Backstory as clue source** (especially investigation
campaigns): PC's background lets them recognise references,
identify suspects, open doors, or interpret evidence others
miss. One of three clue paths, not the only path.

**Faction membership:** faction goals should occasionally
conflict with personal goals; faction NPCs evolve over time;
advancement requires costly choices; faction politics
intersect main arc.

## System-Specific Arc Drivers

### Call of Cthulhu 7e

Arc fuel: SAN loss as progression (what they sacrifice for
truth), personal connections under threat, forbidden knowledge
dilemmas (how far will you go?), backup investigator
foreshadowed as NPC in current story.

Per-session touchpoints: occupation-connected clue, personal
NPC drawn into scenario, Drive-compelled moment despite
danger, sanity test reflecting specific fears.

Ref: GUMSHOE Drive mechanic (Robin Laws / Pelgrane Press)
for motivation-anchored session planning.

### D&D 5e (2024)

Arc fuel: background provides ideal (belief), bond (what PC
cares about), flaw (vulnerability) — arc tests ideal,
threatens bond, exploits flaw. Personal quests tied to
backstory, patron obligations, subclass identity moments,
tier transitions (5/11/17) as arc milestones.

Per-session touchpoints: encounter for one PC's class/background,
backstory NPC, personal-quest magic item/reward, moral choice
testing ideal or exploiting flaw.

### GURPS 4e

Arc fuel: disadvantages are built-in story hooks — Duty/Sense
of Duty (obligation conflicts), Dependents (threatened NPCs
per frequency roll), Enemy (recurring antagonist), Obsession/
Compulsive/Addiction (escalating internal conflict), Secret
(ticking time bomb), Vow/Code of Honour (moral constraints).
Rotate which disadvantage drives B-plot each session. Point-buy
evolution: buying off disadvantages = arc beat. Skill-based
showcases for unique investments.

Per-session touchpoints: disadvantage activation forcing choice,
encounter requiring specific skill/advantage, consequence from
prior disadvantage decision, opportunity to earn points or buy
off disadvantage.

### Forged in the Dark

Arc fuel: Vice scenes during downtime (design vice purveyors
as recurring NPCs), trauma escalation (behaviour shifts per
trauma type), crew advancement as shared arc, entanglements
targeting specific PCs, long-term project clocks as
self-directed arc progression. Four traumas = natural arc
endpoint/retirement.

Per-session touchpoints: downtime vice/project scene,
entanglement targeting PC relationships, score element testing
specific action ratings, faction move threatening something PC
cares about.

## Session Planning Workflow

7 steps. Total prep: 60-75 min on top of standard prep.

| Step | Time | Action |
|------|------|--------|
| 1. PC Roster Review | 10-15 min | Complete template (above) for every PC |
| 2. Arc Check | 5 min | Review Arc Tracker. Determine A/B/C plot structure |
| 3. Touchpoint Assignment | 10 min | Complete Touchpoint Matrix. ≥1 per PC, B-plot PC gets high-impact |
| 4. Scene Design | 15-20 min | Design scenes delivering touchpoints. Embed personal content in group scenes. Every social/event scene MUST include open interaction window (see `scene-encounter-patterns.md`) |
| 5. Connection Audit | 5 min | B-plot advances personal arc AND main plot? Touchpoints feel organic? Any PC getting only mechanical content? |
| 6. Spotlight Forecast | 5 min | Estimate distribution. Below 15% per PC → redesign |
| 7. Agency + Canon Audit | 10-15 min | **Non-negotiable.** See below |

### Step 7: Agency + Canon Audit

**Player Agency — flag violations:**
- PC name or "the party" as subject of declarative action verb
  ("Varrio approaches" → violation; "If Varrio approaches" → ok)
- PC emotions/thoughts/internal state described
- PC action assumed without conditional language (if/should/may/can)
- Read-aloud using 2nd person emotion/perception verbs
  ("You feel uneasy" → violation; describe objective sensory info only)
- Confrontation scene without ≥2 contingencies (cooperative + hostile)

Ref: `scenario-writing.md` Player Agency Constraints.

**Canon Grounding — flag fabrications:**
- Every NPC detail must trace to a source file
- Contacts/options must be established through prior play
- Unverified claims flagged for GM confirmation, not stated as canon

Ref: `continuity-engine.md` Canon Grounding Check.

## Anti-Patterns

| Anti-Pattern | Problem | Prevention |
|-------------|---------|-----------|
| Protagonist Problem | One PC's arc dominates | Rotation + tracking |
| Backstory Dump | Too much backstory in one session | Dole out across many sessions |
| Forced Connection | Every element tied to every PC's past | Some elements new to everyone |
| Solo Spotlight | Personal scene while others sit idle | Embed in group scenes |
| Neglected Quiet Player | Assuming quiet = doesn't want spotlight | Offer opportunity; provide written notes between sessions |
| Arc Railroad | Predetermining arc resolution | GM creates situations; player decides outcome |
| Stale Arc | No arc advancement for 4+ sessions | Even small C-plot clue keeps arc alive |
