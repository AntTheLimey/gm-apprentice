# Random and Procedural Generation

Frameworks for oracle systems, random tables, name generators,
and emergent content. Covers structured random generation
(curated table entries) and procedural generation (rules
producing novel combinations).

## When to Use

- **Improvisation:** PCs go somewhere unplanned, need content now
- **Inspiration:** GM has creative block, needs a spark
- **Surprise:** GM wants to be surprised by own campaign
- **Volume:** many similar elements needed (taverns, guards, rumours)
- **Fairness:** random events feel impartial vs GM fiat

Don't replace intentional design for narratively critical
content (climactic villain, central mystery, key emotional beats).

## Oracle Systems

### Yes/No Oracle (d6)

| Roll | Result |
|------|--------|
| 1 | No, and... (worse than expected) |
| 2 | No |
| 3 | No, but... (silver lining) |
| 4 | Yes, but... (complication) |
| 5 | Yes |
| 6 | Yes, and... (better than expected) |

**Probability modifier:**

| Likelihood | Yes on... |
|-----------|-----------|
| Very unlikely | 6 only |
| Unlikely | 5-6 |
| Even odds | 4-6 |
| Likely | 3-6 |
| Very likely | 2-6 |
| Nearly certain | Auto yes, roll for "and..." |

### Name Generation

Maintain pre-generated lists by setting/culture. Names should
be: pronounceable at table, distinct (different starting letters
and syllable counts), setting-appropriate, memorable (1-2
syllables for minor, longer for major). Keep 20-30 unused
names ready. Cross off and record assignments immediately.

## Random Encounter Generation

### Layered Encounter Tables

**Layer 1 — Who/What (d12 or d20):** creature, NPC type, or
environmental hazard (campaign-specific).

**Layer 2 — Activity (d8):**

| Roll | Activity |
|------|----------|
| 1 | Fleeing from something |
| 2 | Searching for something |
| 3 | Guarding a location/item |
| 4 | Engaged in ritual/task |
| 5 | Fighting each other |
| 6 | Setting an ambush |
| 7 | Resting or injured |
| 8 | Travelling with purpose |

**Layer 3 — Complication (d6):**

| Roll | Complication |
|------|-------------|
| 1 | Weather/environmental change |
| 2 | Third party arrives |
| 3 | Hazardous terrain |
| 4 | One side has info PCs need |
| 5 | Moral dilemma present |
| 6 | Not what it appears |

**Principles:** weight entries by frequency; ≥30% non-combat
entries; connect to current arc; re-roll/discard results that
break continuity or tone.

## Rumour Tables

Each rumour has: **Content** (what it claims), **Truth Value**
(true / partially true / false), **Source** (who's spreading it
and why).

| Roll | Rumour | Truth | Source |
|------|--------|-------|--------|
| 1 | "The old mill is haunted." | Partial: someone hiding there | Local children |
| 2 | "Mayor owes the thieves' guild." | True | Disgruntled guild member |
| 3 | "Travellers disappearing on north road." | True | Merchant who lost caravan |
| 4 | "New priest is a wanted criminal." | False (but priest has secrets) | Jealous rival |

Design: true rumours → active threads; partial = useful truth
+ misleading falsehood; false = reflect source's fears/prejudices;
avoid deliberate red herrings (players generate their own).

## Procedural Generation Principles

- **Combination > invention:** "fire-breathing spider" more
  evocative than wholly original creature (combines known concepts)
- **Constraint-driven:** "Generate a CoC 7e NPC who works at
  the university, opposes the cult, and connects to the victim"
  >> "Generate an NPC"
- **Serendipity is a feature:** surprising combinations are the
  point; interpret in the most interesting way, don't re-roll
  for predictability
- **Override when:** result contradicts canon, breaks tone,
  creates unsafe situation, or simply isn't fun
