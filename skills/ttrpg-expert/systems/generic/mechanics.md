# Generic System -- Core Mechanics

## Overview

This file covers universal RPG mechanical concepts. Use it as a
fallback when the user's game system doesn't have a dedicated
subfolder. The goal is not to guess their rules but to identify
which mechanical paradigm they're using and adapt advice
accordingly.

When a user says "I'm running [system you don't recognize]," start
here. Ask them to describe their core resolution mechanic and work
from the universal patterns below.

## Common Attribute Patterns

Most RPGs define characters through a set of core attributes.
The number and names vary, but they almost always cover three
domains: physical capability, mental capability, and social or
spiritual capability.

### Major Paradigms

| Pattern | Examples | Typical Range |
|---------|----------|---------------|
| Six attributes (phys/ment/soc x power/finesse) | D&D, Pathfinder, 13th Age | 3-20 |
| Three broad attributes | Savage Worlds (3 from Agility/Smarts/Spirit/Strength/Vigor), many PbtA games | Dice type or -1 to +3 |
| Percentile attributes | Call of Cthulhu, BRP, Warhammer Fantasy | 1-100 |
| Point-based attributes | GURPS, Hero System | 1-20+ |
| Freeform / tag-based | Fate (aspects), Risus (cliches) | Narrative or rated 1-5 |

**What to ask the user:** "What are your system's core
attributes or stats? How many are there, and what range do
they use?"

## Resolution Mechanics

Every RPG has a core resolution mechanic -- the procedure for
determining whether an action succeeds or fails. Identifying
which type the system uses is the single most important step
for giving useful mechanical advice.

### The Major Paradigms

**d20 roll-over (target number)**
Roll a d20, add modifiers, meet or beat a target number (DC, AC,
TN). Used by D&D, Pathfinder, 13th Age, d20 Modern.

**Percentile roll-under**
Roll d100 (or two d10s), try to roll equal to or under your
skill rating. Used by Call of Cthulhu, BRP, Warhammer Fantasy
Roleplay, Eclipse Phase.

**Dice pool -- count successes**
Roll a pool of dice (usually d6s or d10s), count how many meet
a threshold. More successes = better result. Used by World of
Darkness, Shadowrun, Forged in the Dark.

**Dice pool -- highest result**
Roll a pool, take the single highest die. Used by Forged in the
Dark (for action rolls specifically), some OSR games.

**2d6 + stat (Powered by the Apocalypse)**
Roll 2d6, add a stat modifier. 6- is a miss, 7-9 is a partial
success, 10+ is a full success. Used by Apocalypse World,
Dungeon World, Masks, Monster of the Week.

**Step dice**
Attributes and skills are rated as die types (d4, d6, d8, d10,
d12). Roll the appropriate die against a target number or
opposed roll. Used by Savage Worlds, Cortex Prime, Earthdawn.

**Card-based or diceless**
Resolution uses cards, tokens, or pure narrative negotiation.
Used by Dread (Jenga), Amber Diceless, Castle Falkenstein.

### Identifying the System Type

Ask the user: "When your character attempts something risky,
what dice do you roll? What number determines success?" The
answer will slot into one of the paradigms above.

## Success and Failure Spectrums

Not all systems treat outcomes as binary pass/fail. The major
patterns:

| Spectrum | How It Works | Examples |
|----------|-------------|----------|
| Binary | Pass or fail, no middle ground | Basic d20, roll-under |
| Graded (degrees of success) | Margin of success/failure matters | CoC (hard/extreme), Warhammer (SL) |
| Tiered (move results) | Distinct outcomes at each tier | PbtA (miss / partial / full) |
| Yes-and / yes-but / no-but / no-and | Four-tier narrative spectrum | Fate, many narrative games |
| Consequences on success | You succeed but at a cost | FitD (position/effect) |

**GM guidance:** When you don't know a system's failure model,
default to "yes, but" -- the action succeeds with a complication.
This works in virtually any system and keeps play moving.

## Action Economy

Most systems with tactical combat have some form of action
budgeting per turn. Common patterns:

| Pattern | Description | Examples |
|---------|------------|----------|
| Major / minor / reaction | One big action, one small, one response per round | D&D 5e, Pathfinder 2e |
| Action points | Spend from a pool each turn (move = 1, attack = 2, etc.) | GURPS, some tactical games |
| Narrative actions | You do one thing, then the GM responds | PbtA, FitD |
| Tick-based | Actions have a speed cost; initiative cycles by ticks | Exalted, some wargames |
| Round-based (everyone acts once) | Each combatant gets one turn, order determined by initiative | Most traditional RPGs |

**What to ask:** "How many things can a character do on their
turn? Is there a distinction between big and small actions?"

## Modifiers and Bonuses

Most systems have ways to make rolls easier or harder:

- **Flat modifiers:** Add or subtract a number (+2, -4).
- **Advantage / disadvantage:** Roll extra dice and take best or
  worst (D&D 5e, Shadow of the Demon Lord).
- **Bonus / penalty dice:** Add or remove dice from a pool
  (CoC, WoD).
- **Shifted difficulty:** The target number changes rather than
  the roll.
- **Fiction-first positioning:** The narrative determines whether
  a roll is even possible (FitD, PbtA).

## When You Don't Know the Rules

If a user asks about a system you have no reference for:

1. **Don't guess.** Say: "I don't have dedicated rules reference
   for [system]. Let me help you based on universal principles."
2. **Ask the key questions:**
   - What dice do you roll to resolve an action?
   - What determines success (target number, counting successes,
     tiered results)?
   - How are characters defined (attributes, skills, classes,
     playbooks)?
3. **Map to a paradigm.** Once you know the resolution mechanic,
   you can give meaningful advice about probability, difficulty
   calibration, and encounter balance.
4. **Lean on the user's knowledge.** They know their system
   better than you do. Your value is in GM technique, encounter
   design, and narrative structure -- not rules lookup.
