Landing Page Approaches

Three layouts for the campaign dashboard — click to select your preference

A

Top-Down Briefing

Recommended. Full-width hero with campaign image, then content flows vertically in clear sections:

  1. Hero — campaign image, title, in-game date + last played
  2. Story So Far — auto-extracted first paragraph of latest session recap
  3. The Team — PC roster cards (portrait, name, status, traits)
  4. Key NPCs — compact cards for tagged NPCs (patron, villain, companion) with role badge
  5. Explore — collapsed nav grid (category + count + link, no individual listings)

Pros

  • Clean, scannable
  • Works on mobile
  • Mirrors a mission briefing

Cons

  • Long vertical scroll
  • NPCs and PCs in separate sections
B

Two-Column Intel Board

Hero at top, then a two-column layout: left column for narrative (recap + timeline), right column for personnel (PCs + key NPCs). Nav grid below.

  1. Hero — same as A
  2. Left: Situation Report — session recap + in-game date
  3. Right: Personnel — PC roster + key NPC cards interleaved
  4. Full-width: Explore — collapsed nav grid

Pros

  • Dense, info-rich
  • Everything above the fold on desktop

Cons

  • Collapses to single column on mobile anyway
  • More complex template
C

Hybrid — Hero + Tabs

Hero at top, then a tabbed section: "Recap" | "The Team" | "Explore". Only one tab visible at a time.

  1. Hero — same as A
  2. Tab: Recap — session extract + dates
  3. Tab: The Team — PCs + key NPCs together
  4. Tab: Explore — full nav grid

Pros

  • Compact, no scrolling
  • Player picks what they want

Cons

  • Hides content behind clicks
  • Requires JavaScript
  • Harder to scan quickly
